# 孕期体重助手 · 开发 Issue 拆分

> 来源：`docs/产品架构设计.md`  
> 产品原则：极快、极简、无广告、无社区、无打卡排名、无焦虑营销  
> 核心目标：每天打开 10 秒，记录，关闭

---

## 推进方式

建议按 Issue 编号顺序推进。每个 Issue 尽量保持可独立验收，避免一次性做太大。

优先级说明：

| 优先级 | 含义 |
| --- | --- |
| P0 | MVP 必做，完成后可以开始真实打卡 |
| P1 | 完整体验，完成后适合长期自用 |
| P2 | 稳定性与体验增强 |
